# 💰 Finance Manager

> Итоговый проект по дисциплине «Объектно-ориентированное программирование»  
> Язык: Python | Интерфейс: консольный + GUI (tkinter)

---

## 📋 Описание проекта

**Finance Manager** — консольное приложение для учёта личных доходов и расходов. Программа позволяет добавлять финансовые операции, просматривать историю, фильтровать по категориям, анализировать статистику и сохранять данные между сеансами.

| Параметр | Значение |
|---|---|
| Язык программирования | Python 3 |
| Тип интерфейса | Консольный + графический (tkinter) |
| Хранение данных | JSON-файл (`data.json`) |
| Для кого | Студенты, которые хотят вести учёт личных финансов |

---

## 🎯 Цель проекта

Разработать приложение для управления личными финансами с использованием принципов объектно-ориентированного программирования: классов и объектов, наследования, инкапсуляции и полиморфизма.

---

## ✅ Функциональные возможности

- ➕ Добавление дохода с указанием суммы, категории и даты
- ➖ Добавление расхода с указанием суммы, категории и даты
- 📋 Просмотр всех транзакций

## 🔑 Применение принципов ООП

### Инкапсуляция

Данные и методы, связанные с одной сущностью, объединены в классе. Например, класс `FinanceManager` хранит список транзакций и предоставляет все методы для работы с ними: добавление, фильтрацию, подсчёт баланса.

### Наследование

Классы `Income` и `Expense` наследуют от абстрактного класса `Transaction`:

```
Transaction (ABC)
├── Income
└── Expense
```

Базовый класс определяет общие поля (`amount`, `category`, `date`) и метод `get_info()`. Дочерние классы реализуют свою версию `calculate_effect()`.

### Полиморфизм

Метод `calculate_effect()` объявлен абстрактным в `Transaction` и реализован по-разному в дочерних классах:

```python
# Income — доход увеличивает баланс
def calculate_effect(self):
    return self.amount       # +сумма

# Expense — расход уменьшает баланс
def calculate_effect(self):
    return -self.amount      # -сумма
```

Это позволяет единообразно считать баланс:

```python
balance = sum(t.calculate_effect() for t in self.transactions)
```

## 📁 Хранение данных

Приложение сохраняет все транзакции в файл `data.json` в корне проекта. Формат хранения — массив JSON-объектов, каждый из которых содержит тип транзакции, сумму, категорию и дату.

Данные сохраняются вручную через пункт меню **«7. Сохранить»** и загружаются через **«8. Загрузить»**.
